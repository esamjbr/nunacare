# Refactor Planning

Before refactoring:

1. Identify current behavior.
2. Identify existing tests.
3. Suggest characterization tests if needed.
4. Define target design.
5. Split into safe steps.
6. Avoid public behavior changes unless requested.
7. Provide rollback strategy.
8. List files likely to change.

Do not edit until the plan is approved.
